Para falar sobre o futuro dos containers

É preciso viajar um pouco ao passado

A virtualização teve inicio nos anos 60, como um método para dividir logicamente os recursos de sistema providos por um mainframe

Os primeiros computadores eram extremamente caros e tipicamente fabricados para resolver um determinado e caracteristico conjunto de tarefas operadas através de paineis de controle via switches. Estes programas poderiam durar horas ou ate mesmo semanas para rodar. Ao medida que os computadores aumentaram sua velocidadde e o tempo de execução foi ficando menor o tempo para carregar um próximo programa começou a ser uma preocupação. Metodologias de processamento em lote evoluiram afim de resolver esse problema acabando como os "dead periods" ao enfileirar diversos programas. Assim que um programa anterior fosse completado o próximo era iniciado.

Quando o programa finalmente acabava a execução o resultado era imprimido e retornado ao programador. O processo completo durava de dias e talvez o programador nem via o computador. A alternativa de deixar o usuário operar o computador diretamente era muito cara pra ser pensada . Isto era devido o fato de o usuário demorar longos periodos para dar entrada no codigo a ser executado.. E ainda o computador estava ociosio.

Time-sharing foi envisionado para acabar com a ociosidade dos mainframes.


Foi constatado que um numero bem maior de usuários poderiam utilizar os mesmos recursos em conjunto.

O time-sharing, foi um dos pilares da computação ao qual conhecemos hoje, os processos, multiplexadores, threads foram originados do mesmo problema

Mas existem vários tipos de virtualização

  • Hardware virtualization
  • Operating system-level virtualization
  • Partial Virtualization
  • Application virtualization

Benefícios da virtualização total

  • Isolation
  • Eficient resources utilization
  • Failover Capabilities
  • Checkpointing e Live Migration

Container

É uma virtualização a nível de sistema operacional que permite a existência de multiplos e isolados user-spaces

Em outras palavras...

A tecnologia permite o isolamento de recursos com um overhead menor

Uma vez que as chamadas de sistema ainda são feitas pelo mesmo kernel e você não precisa virtualizar todas as camadas

Quando os containers surgiram?

Por que então que eles ficaram tão famosos atualmente?

Docker

Docker é um grande exemplo de como construir uma ferramenta viral e orientado ao desenvolvedor

Algumas possíveis razões da rápida adoção

Portabilidade

Performance

Fácil configuração

Docker Hub App-store

API

  • A developer can figure out what Docker does, install it and do something useful with it in 15 minutes. I first heard this “rule” from Marten Mickos when talking about why MySQL was so successful: low friction to try it out, a simple concept and useful functionality.
  • Docker is a great name and it has a cute logo. It resonates with what the product does and is easy to remember. Engineering-oriented founders sometimes seem to think that names and logos don’t matter if the product is good enough, but a great name can turbocharge adoption and build a valuable brand.
  • The Docker product came from a non-threatening source, a small startup (DotCloud) that was able to broadly partner across the whole industry. If the same product had come from an established enterprise technology player, there would have been much more push-back from that player’s competitors, and the market would probably have split into several competing technologies.

Unikernels

Unikernels são imagens especialiazadas, de enreçamento único contruídas utilizando-se um library operating system

Kernels Specializados

  • MirageOS
  • Rumprun
  • Irmin

Quais são os Benefícios??

Segurança

Ultra velocidade de startup

Unikernel - Demo

docker-machine start unik eval $(docker-machine env unik) docker pull gaetan/dockercraft docker run -ti -p 5555:25565 --name minecraft minecraft
  1. git clone https://github.com/emc-advanced-dev/unik.git

  2. cd unik
  3. make binary
  4. mv _build/unik /usr/local/bin/
  5. docker-machine create -d virtualbox --virtualbox-ui-type "headless" unik
